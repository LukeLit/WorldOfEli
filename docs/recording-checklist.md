# Recording Checklist

Use this checklist so each build session is easy to review later.

## Before Recording

- Close any files that contain secrets (`.env`, tokens, private keys).
- Open only needed windows (repo, terminal, Slack call).
- Confirm mic input levels.
- Start recording and say session date + goal out loud.

## During Session

- Keep short verbal checkpoints:
  - "Idea selected"
  - "Scope locked"
  - "First playable test"
  - "Deployment complete"
- Capture Eli's words about what is fun.
- Keep camera/screen layout consistent.

## After Session

- Save file with consistent naming:
  - `YYYY-MM-DD_world-of-eli_session-XX.mp4`
- Add a short summary in `journal/`.
- Write three bullets:
  - What worked
  - What confused Eli
  - What to improve next time

## Privacy + Safety

- Keep Slack channel private.
- Avoid personal info on screen.
- Do not share recordings publicly without consent.
